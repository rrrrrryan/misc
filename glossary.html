States & patterns how UI behaves

The different faces a component wears, and the tricks that make it feel fast.

Empty · loading · error states

The same view has to handle three realities — "nothing here yet," "still fetching," and "it broke" — and each deserves a designed screen, never a blank. Flip between them:

Skeleton loader

Placeholder shapes in the rough layout of the content that's loading, usually with a shimmer sweeping across — feels faster than a spinner and prevents the page from jumping when real content lands.

Optimistic UI

Update the screen immediately as if the action already succeeded, then reconcile when the server answers — rolling back if it failed. Click the star; tick "simulate failure" to watch it revert.
